# 轮询

- 短轮询
	- 过程
		- 客户端每隔一段时间向服务器发送HTTP请求
		- 服务器接收到请求后立刻响应
	- 优点
		- 实现简单
		- 兼容性强
	- 缺点
		- 浪费带宽
		- 存在延迟
- 长轮询
	- 过程
		- 客户端向服务器发送HTTP请求
		- 服务器接收到请求后，挂起一段时间，直到
			- 数据有更新
			- 超时
		- 客户端接收到响应后，立刻重发请求
	- 优点
		- 实时性强
		- 相比短轮询减少带宽浪费
	- 缺点
		- 服务器压力
- 应用
	- 扫码登录
	- 进度查询

# SSE
## 原理
1. 客户端发起HTTP请求
2. 服务器设置`Content-Type: text/event-stream`，并持续不断的发送响应，每次响应的格式：
	- `event` 事件类型，自定义
	- `data` 数据，格式没有要求，推荐使用`NDJSON`
	- `id` 消息ID，用于重连时标识从哪里开始
	- `retry` 断线后隔多少毫秒后重连
``` json
event: chat-update
data: {"text": "Hello, "}
id: 1

event: chat-update
data: {"text": "how can I "}
id: 2
```
## 实现
### 服务端
``` js
// 1. 建立连接
const source = new EventSource('/api/stream');

// 2. 监听默认事件 (message)
source.onmessage = (event) => {
    console.log('收到数据:', event.data);
};

// 3. 监听自定义事件 (比如服务器发出了 event: userlogon)
source.addEventListener('userlogon', (event) => {
    console.log('用户登录通知:', event.data);
});

// 4. 监听错误（包括连接断开）
source.onerror = (err) => {
    if (source.readyState === EventSource.CLOSED) {
        console.log('连接已关闭');
    } else {
        console.error('发生错误:', err);
    }
};

// 5. 主动关闭连接
// source.close();
```
### 客户端
- `python/flask` 使用生成器
``` python
from flask import Flask, Response
import time

app = Flask(__name__)

@app.route('/api/stream')
def stream():
    def generate():
        # 这里模拟 AI 逐字生成或后台任务进度
        for i in range(10):
            time.sleep(1)
            # SSE 格式严格要求： "data: <内容>\n\n"
            yield f"data: 正在处理第 {i+1} 步...\n\n"
        yield "data: 任务完成！\n\n"

    return Response(generate(), mimetype='text/event-stream')

if __name__ == '__main__':
    app.run(debug=True, port=5000)
```
- `node.js/express`
``` node
const express = require('express');
const app = express();

app.get('/api/stream', (req, res) => {
    // 1. 设置响应头
    res.setHeader('Content-Type', 'text/event-stream');
    res.setHeader('Cache-Control', 'no-cache');
    res.setHeader('Connection', 'keep-alive');
    // 如果有 Nginx 代理，建议加上这个防止缓存
    res.setHeader('X-Accel-Buffering', 'no');

    // 2. 发送初始消息
    res.write('data: 连接已建立\n\n');

    // 3. 模拟定时推送数据
    const timer = setInterval(() => {
        const data = { time: new Date().toLocaleTimeString() };
        res.write(`data: ${JSON.stringify(data)}\n\n`);
    }, 2000);

    // 4. 监听客户端断开连接
    req.on('close', () => {
        clearInterval(timer);
        res.end();
        console.log('客户端已断开');
    });
});

app.listen(3000, () => console.log('SSE server running on port 3000'));
```
## 特点
- 单向传输
	- 适用于流式传输，如AI响应
	- 不适合全双工通信，如聊天室
- 协议轻量
	- 与WebSocket相比不需要协议升级
- 自动重连
	- 内置自动重连机制，不需要手动实现
