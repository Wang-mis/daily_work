# 进程/线程/协程
- 进程
	- 资源分配的基本单位
	- 进程拥有独立的内存空间
	- 进程间通过管道、共享内存、信号量、消息队列等机制通信
- 线程
	- CPU调度的基本单位
	- 同一进程的不同线程间共享它们所属进程的内存空间
	- 线程由于共享内存，可以直接读写同一份内存空间
		- 使用同步和互斥机制防止竞态条件
- 协程
	- **用户态**、**轻量级**的线程
	- 由**用户程序**控制，与操作系统内核无关
	- 切换协程时不需要进行用户态和内核态之间的切换，不需要系统调用，开销最低

# 单线程/多线程
- 多线程优势
	- 充分利用多核，提高运行效率
- 多线程劣势
	- 需要引入锁机制
		- 增加了编程复杂性
		- 加锁/解锁有额外开销
		- 可能出现死锁问题
	- 线程切换上下文有额外开销

# 线程切换
1. 触发时机
	- 时间片结束
	- 主动挂起（sleep等）
	- 阻塞等待（I/O、等待锁等）
	- 被抢占
2. 保存上下文
	- 保存程序计数器（PC/IP）、栈指针（SP）、通用寄存器
3. 调度算法执行
	- 操作系统根据调度算法，选择处合适的待运行的线程
4. 恢复上下文
5. 切换到新线程执行
